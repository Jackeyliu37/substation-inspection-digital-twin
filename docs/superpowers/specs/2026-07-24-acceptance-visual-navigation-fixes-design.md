# 验收视觉与导航修复设计

## 目标

修复自动巡检在首个检查点反复取消的问题和仿真场景命令偶发超时；让三维孪生可旋转、缩放、平移并展示场景效果；让二维地图保持等比、标记清晰且坐标不漂移；将实时视频左转 90°；用四组真实训练预测图展示模型效果。

## 方案

- 任务稳定性：风险内容和队列顺序均未改变时，`MissionEngine.apply_risk` 返回未变化，任务管理器不增加队列版本、不取消 Nav2 goal。真实风险变化仍触发一次重排。
- 场景终态：Gateway 在调用 ROS 参数服务前记录场景版本，以此作为命令期望基线，避免快速终态与命令落库之间的竞态。
- 三维交互：基于现有 React Three Fiber 和 Three.js 实现轻量轨道相机，不新增依赖。左键拖动旋转，右键或 Shift+拖动平移，滚轮缩放，按钮提供旋转、缩放和复位。
- 三维场景：将 Gateway 场景快照传给孪生视图，以明确的火焰/烟雾、气体云、人员、障碍物和仪表光环表示已激活场景，并显示当前场景条。
- 二维地图：栅格和 SVG 叠加层共用同一等比容器及变换；修正 Y 轴一像素差；地图内只保留编号标记和悬停名称，完整设备信息放在侧栏。
- 感知展示：实时 JPEG 使用 CSS 左转 90°并保持完整可见；删除阈值豁免字样；从四个训练测试 run 各提取三张 `val_batch*_pred.jpg`，每个模型独立轮播。

## 验收

- 相同风险快照不会增加 `queue_revision`，连续巡检能越过首检查点。
- 连续触发多个场景均在 8 秒内得到成功终态，三维页面展示对应效果。
- 地图保持 4:3 栅格比例，设备、路线、目标和机器人共享坐标变换，不显示重叠长标签。
- 三维视图支持拖拽、滚轮及按钮控制；感知视频方向正确；四组模型轮播素材来自上传的训练结果。
