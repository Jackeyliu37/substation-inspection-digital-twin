# 地图、导航与驾驶舱细节修复设计

## 目标

让二维地图准确表达全部十台设备、Nav2 当前规划路径和机器人实际轨迹，并消除自动巡检启动后多个导航目标互相取消导致的抖动停滞。

## 设计

- Gateway 订阅 Nav2 `/plan`，只接受 `map` 坐标系且数值有限的 `nav_msgs/Path`，将点列随 `/api/v1/map` 返回。
- 前端分别显示“Nav2 实时规划路径”和“机器人已行驶轨迹”，不再把全部任务目标用直线串联。
- 十台资产都按类别显示矩形设备轮廓；编号使用确定性的碰撞避让与引线，避免近邻设备编号重叠。
- 任务管理器在已有执行目标时，周期状态发布不得触发替换；只有紧急风险显式请求替换。运行时恢复时把无法跨进程延续的 `active` 任务恢复为 `queued`。
- 驾驶舱资产列表滚动条使用黑色窄轨道和深灰滑块。

## 验证

- ROS 单元测试覆盖周期发布不取消活动目标、恢复快照清理活动任务、执行反馈保持单一活动任务。
- Gateway 测试覆盖 Nav2 路径校验与地图投影。
- 前端契约测试覆盖设备轮廓、标签避让、实时规划路径、实际轨迹和黑色滚动条；执行 Next.js 生产构建。
