# 人工验收控制台可用性设计

## 目标

让第一次打开 `http://ros-server/` 的中文用户无需理解 ROS 工程枚举即可完成巡检验收：看懂设备与状态、恢复紧停、启动自动巡检、观察机器人移动，并在二维地图中缩放、拖拽和旋转查看细节。

## 已确认根因

- 激活发布时安全流程会锁存紧急停止；解除紧停前必须停止旧任务并保持至少 0.5 秒零速度。
- 解除紧停后机器人进入手动模式。现有页面没有切换自动模式的入口，因此任务可显示“运行中”，但 Nav2 速度会被安全仲裁器拒绝。
- 现场通过正式 API 切换为自动模式后，机器人 10 秒移动 0.98 米，证明 Gazebo、Nav2、底盘与位姿链路正常。

## 推荐方案

### 中文展示

新增集中式中文映射模块，覆盖资产 ID、设备类别、风险等级、任务状态、机器人模式、场景、模型名称、事件类型和常见后端错误。主要信息显示中文；原始 ID 仅作为小号诊断信息保留。页面不再显示 `OPERATIONS`、`Gateway`、`revision`、`GAZEBO SCENARIO` 等面向开发者的英文标题。

### 自动巡检恢复

驾驶舱提供一个主操作“恢复并开始自动巡检”。它根据实时状态顺序执行：

1. 若紧停锁存且任务仍活动，停止任务并等待状态收敛；
2. 等待零速度安全窗口后解除紧停，遇到安全栅栏尚未就绪时有限重试；
3. 若任务未运行，启动默认巡检路线并等待新运行上下文；
4. 获取最新任务版本和紧停版本，切换到自动模式；
5. 轮询确认机器人为自动模式并刷新页面快照。

页面同时保留单独的暂停、继续、停止和紧急停止按钮。紧停锁存时显示中文原因及恢复步骤。命令失败显示中文说明，不直接暴露英文后端消息。

### 可交互地图

占据地图的栅格层和 SVG 标注层放入同一变换容器，确保路线、设备和机器人始终同步。交互包括：

- 鼠标滚轮缩放，范围 0.75～6 倍；
- 鼠标/触控拖拽平移；
- 按钮每次左右旋转 15°；
- 放大、缩小、复位按钮；
- 显示当前倍率与角度。

不新增前端依赖。变换计算放入纯 JavaScript 模块并用 Node 契约测试覆盖。

### 字体与验收引导

桌面端正文基准提高到 16px，小字不低于 13px，按钮不低于 15px，标题和关键数字按比例放大。地图侧栏加入明确验收步骤：恢复并启动、观察模式为自动、确认坐标/轨迹持续变化、触发场景并复位。

## 验收标准

- 页面主要可见文字均为中文，工程 ID 只作为次要诊断信息。
- 紧停状态可通过一个主操作恢复并进入自动巡检。
- 自动模式下机器人位姿持续变化，二维地图和三维孪生同步更新。
- 地图滚轮缩放、拖拽、左右旋转和复位均有效，叠加层无错位。
- 前端契约测试、Next.js 生产构建及既有 Gateway/集成测试通过。
