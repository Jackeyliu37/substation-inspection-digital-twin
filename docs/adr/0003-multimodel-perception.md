# ADR-0003：保持独立的多模型感知模块

- 状态：Accepted
- 日期：2026-07-22

## 背景

公开数据没有一个来源能同时可靠覆盖变电站设备、人员/PPE、烟火、设备缺陷和仪表读数；检测框、设备裁剪分类和表盘几何读数也不是同一任务。项目需要可独立训练、评估和追溯的 YOLO11n 感知能力，并将温度、气体和上下文风险与视觉结果在风险层融合。仪表数据唯一来源是 Gazebo 自动生成的合成数据。

## 已考虑的方案

1. **四个独立模块（选择）。** YOLO11n 安全检测、YOLO11n 设备检测、YOLO11n 缺陷分类，以及 YOLO11n 表盘定位加 OpenCV 读数，各自训练、评估和发布结果。
2. **将所有类别和任务合并为一个未经评估的大检测/多任务模型。** 数据域、类别不均衡和输出语义不同，难以证明小类、缺陷分类和仪表读数均合格。
3. **用规则、场景真值或外部仪表数据替代独立视觉模块。** 场景真值会把答案泄漏给推理；外部仪表数据违反本期只使用 Gazebo 合成仪表数据的约束。

## 决定

保持下列独立责任：

- 安全检测：`yolo11n_safety.pt`，关注人员、PPE、火焰和烟雾；
- 设备检测：`yolo11n_equipment.pt`，覆盖固定的变电站设备类别；
- 缺陷分类：`yolo11n_fault.pt`，对独立设备裁剪图输出正常和故障状态；
- 仪表读数：YOLO11n 定位表盘，OpenCV 进行透视、圆盘/刻度/指针分析，仅用 Gazebo 合成仪表图训练与验证。

这些模块只向数字孪生提供带来源和置信度的观测。风险引擎在配置的权重、连续帧确认、滑动平均和滞回下，与温度、烟雾/气体和上下文风险融合；模型不得读取 Gazebo 场景真值。

## 后果

- 每一模块可拥有独立数据 manifest、类别映射、权重 SHA-256、指标和回归测试，失败不会被整体指标掩盖。
- 训练、部署和监测需要维护多组模型与结果模式；Gateway 和报告必须保留观测来源而非将其伪装为统一模型输出。
- 某个视觉模块不可用时，系统明确记录该证据缺失并按架构降级，而不是把缺失结果解释为正常。
- 新模型、数据许可或版本变化必须通过 ADR、版本锁定、数据治理和验收基线的同步变更。

## 可以被取代的条件

只有在新方案使用可审计、许可兼容且不泄漏 Gazebo 真值的数据，且在独立保留测试集和目标合成场景上逐项达到或超过安全检测、设备检测、缺陷分类和仪表读数各自的验收指标，并证明风险融合、延迟、降级和可追溯性不退化时，才可新建一个 Superseding ADR。项目计划、数据/模型治理、接口、版本锁定、权重校验和测试基线必须先或同步更新。单一总体 mAP 提升、训练成本降低或未评估模型宣传不足以取代本决定。
